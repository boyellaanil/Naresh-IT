HTML

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a webpage using elements and tags.

HTML elements are represented by tags, which are enclosed in angle brackets. For example, the <h1> tag is used for main headings, while the <p> tag is used for paragraphs.

HTML documents can include text, images, links, and other multimedia content, making it a versatile language for web development.

Elements and Tags

HTML elements are the building blocks of HTML pages. An element consists of a start tag, content, and an end tag.

For example, the following is a simple HTML element:

            <tagname>Content goes here</tagname>

In this example, tagname is the name of the element, and "Content goes here" is the content of the element.

Some elements can be self-closing, meaning they do not require an end tag. For example:

            <img src="image.jpg" alt="Description">

In this case, the <img> tag is self-closing and does not have a separate end tag.

HTML tags are not case-sensitive, meaning <TAGNAME> and <tagname> are treated the same. However, it is a common practice to use lowercase for consistency.

HTML elements can also have attributes, which provide additional information about the element. Attributes are specified within the start tag and consist of a name and a value, like this:

            <tagname attribute="value">Content goes here</tagname>

For example, the <a> tag (anchor tag) can have an href attribute to specify the URL of the link:

            <a href="https://www.example.com">Visit Example</a>

In this example, the href attribute specifies the URL that the link points to.

Comments

Comments in HTML are used to add notes or explanations within the code that are not displayed in the browser. They are useful for developers to document their code.

Comments are written using the following syntax:

                <!-- This is a comment -->

Anything between the <!-- and --> will be treated as a comment and will not be rendered in the browser.

For example:

                <!-- This is a comment and will not be displayed in the browser -->
                <p>This is a paragraph.</p>

In this example, the comment will not be visible on the webpage, but it can help other developers understand the purpose of the code.

Comments can span multiple lines as well:

                <!--
                This is a multi-line comment.
                It can span multiple lines.
                -->

Using comments effectively can improve code readability and maintainability, especially in larger projects.

            <h3>Sub-subheading</h3>

            <h4>Fourth Level Heading</h4>

            <h5>Fifth Level Heading</h5>

            <h6>Sixth Level Heading</h6>

In the above code:

            <h1> - Represents the main heading of the document.

            <h2> - Represents a subheading under the main heading.

            <h3> - Represents a sub-subheading under the subheading.

Headings are important for structuring content and improving accessibility. They help search engines understand the hierarchy of information on a webpage.

It is recommended to use headings in a logical order, starting with <h1> for the main title and using lower-level headings as needed. Avoid skipping heading levels (e.g., going from <h1> to <h3> without using <h2>).

Anchor Tag

The anchor tag is used to create hyperlinks. It is represented by the <a> tag.

Example:
<a href="https://www.example.com">Visit Example</a>

The href attribute specifies the URL of the page the link goes to.

When clicked, the link will take the user to the specified URL.

Anchor tags can also be used to link to specific sections within the same page using the "#" symbol followed by the ID of the target element. For example:

<a href="#section1">Go to Section 1</a>
...
<h2 id="section1">Section 1</h2>

In this example, clicking the link will scroll the page to the element with the ID "section1".

Anchor tags can also be used to create email links using the "mailto:" protocol. For example:

<a href="mailto:abc@gmail.com">Send Email</a>

In this case, clicking the link will open the user's default email client with a new message addressed to abc@gmail.com.

You can also add a subject and body to the email link:

<a href="mailto:abc@gmail.com?subject=Hello&body=How are you?">Email with Subject</a>

To open links in a new tab, use the target attribute with the value _blank:

<a href="https://www.google.com" target="_blank">Open Google in new tab</a>

To improve security and SEO when using target="_blank", add rel="noopener noreferrer":

<a href="https://www.google.com" target="_blank" rel="noopener noreferrer">Secure New Tab Link</a>

Images

Images can be added to an HTML document using the <img> tag.

            <img src="image.jpg" alt="Description of image">

The "src" attribute specifies the path to the image file, and the "alt" attribute provides alternative text for the image.

For example:

            <img src="https://www.example.com/image.jpg" alt="A beautiful scenery">

In this example, the image will be displayed on the webpage, and if the image cannot be loaded, the alternative text "A beautiful scenery" will be shown instead.

Images can also be resized using the "width" and "height" attributes:

            <img src="image.jpg" alt="Description of image" width="300" height="200">

In this case, the image will be displayed with a width of 300 pixels and a height of 200 pixels.

It is important to use the "alt" attribute for accessibility purposes, as it provides a text alternative for users who cannot see the image, such as those using screen readers.

Div and Span

The <div> and <span> tags are used for grouping content and applying styles.

The <div> tag is a block-level element, while the <span> tag is an inline element.

            <div>

                <p>This is a paragraph inside a div.</p>

            </div>

            <span style="color: red;">This is red text.</span>

The "style" attribute can be used to apply CSS styles directly to these elements.

For example, you can use the <div> tag to create a section of a webpage:

            <div class="container">

                <h2>Section Title</h2>

                <p>This is a section of content.</p>

            </div>

In this example, the <div> tag is used to group a heading and a paragraph together, creating a section of content that can be styled as a whole.

The <span> tag is often used to apply styles to a specific part of text within a paragraph or other inline content. For example:

            <p>This is a <span style="font-weight: bold;">bold</span> word in a sentence.</p>

In this case, the word "bold" will be displayed in bold font within the paragraph.

Using <div> and <span> tags effectively can help organize and style your HTML content, making it easier to read and maintain.

Lists

HTML supports both ordered , unordered lists,definition lists

Unordered List

            <ul>

                <li>Item 1</li>

                <li>Item 2</li>

                <li>Item 3</li>

            </ul>

In the above code:

            <ul> - Starts an unordered list.

                <li> - Defines a list item.

            </ul> - Ends the unordered list.

Unordered lists are typically displayed with bullet points.

Ordered List

            <ol>

                <li>First Item</li>

                <li>Second Item</li>

                <li>Third Item</li>

            </ol>

In the above code:

            <ol> - Starts an ordered list.

                <li> - Defines a list item.

            </ol> - Ends the ordered list.

Ordered lists are typically displayed with numbers.

Both unordered and ordered lists can be nested within each other to create sublists. For example:

            <ul>

                <li>Item 1</li>

                <li>Item 2</li>

                <li>Sublist:<ul>

                    <li>Subitem 1</li>

                    <li>Subitem 2</li>

                </ul></li>

            </ul>

In this example, the sublist is nested within the second list item of the main unordered list.

Ordered List with Custom Start

            <ol start="5">

                <li>Fifth Item</li>

                <li>Sixth Item</li>

            </ol>

In the above code, the start attribute is used to specify that the ordered list should start at item 5 instead of the default 1.

Definition Lists

            <dl>

                <dt>Term 1</dt>

                <dd>Definition of Term 1</dd>

                <dt>Term 2</dt>

                <dd>Definition of Term 2</dd>

            </dl>

In the above code:

            <dl> - Starts a definition list.

                <dt> - Defines a term.

                <dd> - Defines the description or definition of the term.

            </dl> - Ends the definition list.

Definition lists are used to display terms and their definitions, often used in glossaries or dictionaries.

Lists are useful for organizing content and making it easier to read. They can be styled using CSS to change their appearance, such as bullet styles, numbering formats, and indentation.

Tables

HTML tables are used to display data in a tabular format. They are created using the "<table>" tag, along with "<tr>" for table rows, "<th>" for table headers, and "<td>" for table data cells.

            <table>

                <tr>

                    <th>Header 1</th>

                    <th>Header 2</th>

                </tr>

                <tr>

                    <td>Data 1</td>

                    <td>Data 2</td>

                </tr>

            </table>

In the above code:

            <table> - Starts the table.

                <tr> - Starts a new table row.

                    <th> - Defines a table header cell.

                    <td> - Defines a table data cell.

            </table> - Ends the table.

Tables can have multiple rows and columns, and you can use the "<thead>", "<tbody>", and "<tfoot>" tags to group the header, body, and footer sections of the table.

For example:

            <table>

                <thead>

                    <tr>

                        <th>Header 1</th>

                        <th>Header 2</th>

                    </tr>

                </thead>

                <tbody>

                    ...

            </tbody>

            </table>

In this example, the "<thead>" section contains the table headers, while the "<tbody>" section contains the main data rows.

Tables can also have a caption using the "<caption>" tag:

            <table>

                <caption>Table Caption</caption>

                <tr>

                    <th>Header 1</th>

                    <th>Header 2</th>

                </tr>

            </table>

In this case, the caption will be displayed above the table, providing a title or description for the table's content.

Forms

HTML forms are used to collect user input. They are created using the <form> cotent </form> tag, along with various input elements such as text fields, checkboxes, radio buttons, and submit buttons.

            <form action="/submit" method="post">

                <label for="name">Name:</label>

                <input type="text" id="name" name="name">

                <input type="submit" value="Submit">

            </form>

In the above code:

            <form> - Starts the form.

                <label> - Defines a label for an input element.

                <input> - Defines an input field.

                <input type="submit"> - Defines a submit button.

            </form> - Ends the form.

The "action" attribute specifies the URL where the form data will be sent when the form is submitted, and the "method" attribute specifies the HTTP method to use (e.g., "get" or "post").

Forms can include various input types, such as:

  • text - Single-line text input
    • <input type="text" name="username">
  • password - Password input (hidden characters)
    • <input type="password" name="password">
  • email - Email input with validation
    • <input type="email" name="email">
  • number - Numeric input with validation
    • <input type="number" name="age">
  • checkbox - Checkbox input
    • <input type="checkbox" name="subscribe">
  • radio - Radio button input
    • <input type="radio" name="gender" value="male">Male
  • submit - Submit button
    • <input type="submit" value="Submit">
  • reset - Reset button to clear form
    • <input type="reset" value="Reset">
  • color - Color picker input
    • <input type="color" name="favcolor">
  • date - Date selector
    • <input type="date" name="birthdate">
  • range - Range slider
    • <input type="range" name="volume" min="0" max="100">
  • file - File upload selector
    • <input type="file" name="resume">
  • search - Search field input
    • <input type="search" name="query">
  • tel - Telephone number input
    • <input type="tel" name="phone">
  • url - URL input field
    • <input type="url" name="homepage">

For example, to create a form with a text input and a submit button:

            <form action="/submit" method="post">

                <label for="username">Username:</label>

                <input type="text" id="username" name="username">

                <input type="submit" value="Submit">

            </form>

In this example, the form includes a text input for the username and a submit button. When the user fills in the username and clicks the submit button, the form data will be sent to the specified action URL.

Forms can also include additional features such as validation, required fields, and more advanced input types. For example, you can use the "required" attribute to make an input field mandatory:

            <form action="/submit" method="post">

                <label for="email">Email:</label>

                <input type="email" id="email" name="email" required>

                <input type="submit" value="Submit">

            </form>

In this case, the email input field is marked as required, meaning the user must fill it out before submitting the form.

Forms can also include dropdown menus, text areas, and other input types to collect more complex data. For example:

            <form action="/submit" method="post">

                <label for="country">Country:</label>

                <select id="country" name="country">

                    <option value="usa">USA</option>

                    <option value="canada">Canada</option>

                </select>

                <input type="submit" value="Submit">

            </form>

In this example, the form includes a dropdown menu for selecting a country, along with a submit button. When the user selects a country and clicks the submit button, the selected value will be sent to the specified action URL.

Forms are essential for user interaction on websites, allowing users to submit data, register accounts, search for information, and perform various actions. They can be styled using CSS to improve their appearance and usability.

In this action can have two methods

  • get - Sends form data as URL parameters (visible in the URL).
  • post - Sends form data in the request body (not visible in the URL).

For example, to use the "get" method:

            <form action="/search" method="get">

                <label for="query">Search:</label>

                <input type="text" id="query" name="query">

                <input type="submit" value="Search">

            </form>

In this example, when the user submits the form, the search query will be appended to the URL as a query parameter (e.g., "/search?query=your_search_term").

Using the "post" method is more secure for sensitive data, as it does not expose the data in the URL. For example:

            <form action="/submit" method="post">

                <label for="password">Password:</label>

                <input type="password" id="password" name="password">

                <input type="submit" value="Submit">

            </form>

In this case, the password will be sent in the request body when the form is submitted, keeping it hidden from the URL.

HTML 5

HTML5 is the latest version of HTML, which introduces new features and elements for modern web development. Some key features include:

  • New semantic elements like <header>,<footer>, <article> and <section>.
  • Support for multimedia elements like <video> and <audio>.
  • Improved form controls and input types.
  • Canvas element for drawing graphics.

Semantic Elements

Semantic elements provide meaning to the content they contain, making it easier for browsers and search engines to understand the structure of a webpage. Examples include:

  • <header> - Represents the header of a document or section.
  • <footer> - Represents the footer of a document or section.
  • <article> - Represents a self-contained piece of content.
  • <section> - Represents a thematic grouping of content.
  • <nav> - Represents a navigation section.
  • <aside> - Represents content that is tangentially related to the main content.
  • <main> - Represents the main content of a document.

Multimedia

HTML5 provides native support for multimedia content, allowing developers to embed audio and video directly into web pages without the need for third-party plugins.

Both audio and video elements can include multiple source files to ensure compatibility with different browsers and devices.

Iframes

An iframe (inline frame) is used to embed another HTML document within the current document. It is represented by the <iframe> </iframe> tag.

            <iframe src="https://www.example.com" width="600" height="400"></iframe>

The src attribute specifies the URL of the document to be embedded, while the width and height attributes define the size of the iframe.

Using iframes, you can display content from other websites or applications within your own webpage, allowing for a seamless user experience.
